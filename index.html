File Server   In my operating systems class, I worked in a group to create a multi-threaded file server. It was built to maximize the amount of concurrent reads and writes. We built the client to server protocol from scratch! We created the message system, broke it down to bits, then used sockets to send the message, and used error checking on both sides as we reinterpretted the bits. To traverse the file system we created, we used hand over hand locking to safetly maximize concurrency and provide an interface of read, write, create, and delete files to the clients.
Distributed System Sockets Client Server Paradigm Hand Over Hand Locking Multi threading C++

Thread Scheduler   In my operating systems class, we created a thread and lock API. This was a challenging task, because we had to save the stack of each thread, move it around, then put it back on the CPU when the CPU became available. We built up locks and threads from just a single atomic bool switch statement and linux context savers. This project made me internalize and deeply understand how threads and locks work. It was the most difficult project I've ever done, but it taught me so much about debugging, multi-threading debugging, code design, and testing.
Debugging Multi threading Synchronization C++

Hadoop Map Reduce   In my web systems class, we created a distributed master worker paradigm to create an efficient map reduce program. The master listened for queries from the client, and then distributed the work to the workers. The workers were created from a pool of threads, to avoid the overhead of creating and destroying threads. We also made the map reduce fault tolerant by implementing a heart beat system between workers to master. The heartbeat paradigm can be found in google file system and many other large system projects.
Distributed System Master Worker Paradigm Heartbeat Paradigm Sockets Multi threading Python
