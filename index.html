Network File Server   In my operating systems class, I worked in a group to create a synchronized, fault tolerant file server. To traverse the file system, we used hand over hand locking to safetly maximize concurrency. We provide an interface of read, write, create, and delete files to the clients. We gracefully managed a file exceeding the size of it's disk block by maintaining disk data structures that mapped where all the memory of a file lived. We also built a secure, robust client to server protocol from scratch! The client server protocol involved creating the message, breaking it down to bits, encryping the message, sending the message using Berkley sockets, and robustly error checking the message on the other side.
Distributed System Sockets Client Server Paradigm Hand Over Hand Locking Multi threading Assert Statements C++

Thread Library   In my operating systems class, we created a multicore compatible thread, mutex, and cv library. We built up the library from a test-and-set instruction and linux context managers. We spent a week carefully thinking about and designing the OS datastructures needed to manage this project. The data structures rougly involve maintining queues for CPUs, mutexes, and cv's along with managing thread contexts. The library is extremely robust, and is able to gracefully handle CPU interrupts at any point in time. We learned to use a copious amount of assert statements to catch bugs early, and smart print statements to make sense of underterministic code. It was the most difficult project I've ever done, but it taught me so much about debugging, multi-threading debugging, code design, and testing. I loved the challenge of this project and it made me deeply understand how threads and locks work.
Multi Threading Debugging Multi threading Synchronization Linux Context Managers Assert Statements Threads Mutexes CV's C++

Hadoop Map Reduce   In my web systems class, we created a distributed master worker paradigm to create an efficient map reduce program. The master listened for queries from the client, and then distributed the work to the workers. The workers were created from a pool of threads, to avoid the overhead of creating and destroying threads. The communication from the master to worker was done using python sockets. We also made the map reduce fault tolerant by implementing a heart beat system between workers to master. The heartbeat paradigm can be found in google file system and many other large system projects.
Distributed System Master Worker Paradigm Heartbeat Paradigm Sockets Multi threading Python
